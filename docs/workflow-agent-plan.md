# 工作流创建 Agent 功能开发方案

## 目标

用户用自然语言描述一个可复用创作场景后，系统自动生成一份可编辑的创作工作流草稿。草稿包含基础信息、公开/个人范围、输入变量、变量类型、选项、默认值、系统提示词、提示词模板、负面约束和生图参数。用户确认后保存为正式工作流。

## 典型输入

用户可以粘贴类似 `工作流生成智能体参考提示词.md` 的完整提示词，也可以只写简短需求：

- “帮我创建一个女性写真工作流，可选择写真风格、五官方向、服装、场景、光线和画幅。”
- “创建一个电商海报工作流，只需要输入产品名称、卖点、活动信息。”
- “创建一个小红书封面工作流，风格选项包括美食/旅行/穿搭。”

## 输出结构

Agent 输出必须是结构化 JSON，前端不直接采纳自由文本：

```json
{
  "name": "女性写真提示词生成",
  "category": "写真创作",
  "description": "通过固定风格规则和变量参数生成写真提示词。",
  "scope": "private",
  "variables": [
    {
      "key": "photo_style",
      "label": "写真风格",
      "type": "select",
      "required": true,
      "defaultValue": "清冷私房",
      "options": ["自动", "清冷私房", "轻性感氛围"]
    }
  ],
  "config": {
    "systemPrompt": "...",
    "promptTemplate": "... {{photo_style}} ...",
    "negativePrompt": "...",
    "apiMode": "responses",
    "size": "3:4",
    "quality": "high",
    "count": "1"
  }
}
```

## 功能入口

第一期建议放在生图工作台右侧“创作工作流”抽屉内：

- `新建工作流` 保留手动创建。
- 新增 `AI 创建` 按钮。
- 点击后打开 Agent 弹窗，左侧输入需求，右侧展示生成的工作流草稿。
- 用户可以复制、重新生成、应用到编辑器、保存。
- Agent 弹窗可单独选择文本模型和渠道，不强制使用当前生图模型。
- Agent 弹窗可上传本地参考图，也可以从“我的素材”选择文本或图片作为参考。

## Agent 生成流程

1. 用户输入自然语言需求或粘贴参考提示词。
2. 前端调用后端 `/api/v1/workflows/agent-draft`。
3. 后端使用用户在 Agent 弹窗中选择的文本模型和渠道；本地直连模式会随请求携带对应渠道的 Base URL 与 API Key，云端模式会携带后台渠道 ID。
4. 服务端要求模型只返回 JSON，并执行 JSON Schema 校验。
5. 后端做安全清洗：
   - 变量 key 只允许 `a-zA-Z0-9_-.`。
   - 变量类型限制为 `text / textarea / number / select / boolean`。
   - 选项最多 50 个，单项最多 80 字。
   - 提示词模板必须包含至少一个变量或确认是无变量模板。
6. 前端展示草稿，用户确认后调用现有工作流保存接口。

## 变量识别规则

Agent 需要优先从用户文本中的枚举结构识别变量：

- `{自动 / 清冷私房 / 轻性感氛围}` 自动识别为 `select`。
- 多行字段说明自动识别为变量 label。
- “是否、开启、关闭、需要、不需要” 倾向识别为 `boolean`。
- “数量、比例、强度、年龄、宽度、高度” 倾向识别为 `number` 或 `select`，取决于是否有枚举。
- 长段自由描述识别为 `textarea`。

## 后端接口设计

新增接口：

- `POST /api/v1/workflows/agent-draft`

请求：

```json
{
  "prompt": "用户输入的工作流需求",
  "scope": "private",
  "model": "gpt-5.5",
  "channelId": "channel_xxx",
  "channelMode": "remote",
  "baseUrl": "",
  "apiKey": "",
  "references": ["data:image/png;base64,..."]
}
```

响应：

```json
{
  "draft": {
    "name": "...",
    "category": "...",
    "description": "...",
    "scope": "private",
    "variables": [],
    "config": {}
  },
  "warnings": ["部分变量没有默认值"]
}
```

## 安全与权限

- Agent 只生成草稿，不自动公开发布。
- 默认 `scope=private`，用户必须手动切换公开。
- 公开工作流保存时记录创建者，只有创建者可以编辑和删除。
- 公开工作流被其他用户复制后变成个人副本。

## 分期

第一期：

- Agent 弹窗。
- 生成结构化工作流草稿。
- JSON 校验和变量类型识别。
- 应用草稿到现有工作流编辑器。
- 独立选择不同渠道的文本模型。
- 支持参考图和“我的素材”作为 Agent 输入。

第二期：

- 支持从参考提示词文件导入并自动拆变量。
- 支持“一键重新组织变量顺序”。
- 支持保存为公开模板前的质量检查。

第三期：

- Agent 根据历史生成结果反推工作流。
- 支持多步骤工作流，例如先生成提示词，再生成图片。
