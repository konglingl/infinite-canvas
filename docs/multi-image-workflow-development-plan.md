# 多图生成工作流开发文档

本文档记录 v0.3.0 新增的“多图生成工作流”设计、实现范围和使用方式。该功能用于同一主题下的系列图片创作，例如小红书/公众号配图、多张电商海报、软件宣传图、人物写真组图、店铺装修方案等。

## 已实现状态

v0.3.0 已完成第一期和第二期主要能力：

- 工作流新增 `mode` 字段，支持 `single_image` 和 `multi_image_series`。
- 工作流新增 `seriesConfig`，可配置提示词生成模型/渠道、目标张数、并发上限、审核开关和系列拆分说明。
- 编辑弹窗可在“单图生成”和“多图生成”之间切换。
- 运行弹窗支持先生成多条图片提示词草稿，再逐条或批量生成图片。
- 多图草稿支持编辑、复制、删除、上下移动、批量追加要求、失败重试和本地持久化。
- 工作流列表支持分类筛选，并内置“小红书文章配图组”多图模板。
- AI 创建工作流可识别多图/系列/组图意图，并输出多图工作流草稿。
- 图片结果继续写入生图历史，并自动归入工作流同名分类。

## 核心流程

1. 用户创建或选择一个“多图生成”工作流。
2. 用户填写变量，例如文章主题、产品名称、卖点、风格、目标平台、张数。
3. 系统调用 `seriesConfig.promptModel` 对应的文本模型生成多条图片提示词草稿。
4. 用户人工审核、编辑、删除或排序草稿提示词。
5. 用户点击单条“生成”，或点击“全部生成”按并发上限批量生成图片。
6. 图片生成使用工作流生成配置里的图片模型、渠道、API 模式和生图参数。
7. 结果写入生图历史，带上工作流名称、系列标题、序号、提示词、耗时和参数信息。

## 数据结构

### 工作流模式

```ts
type WorkflowMode = "single_image" | "multi_image_series";
```

### 多图配置

```ts
type WorkflowSeriesConfig = {
  targetCount: string;
  promptModel: string;
  promptChannelId: string;
  promptInstruction: string;
  reviewRequired: boolean;
  concurrency: string;
};
```

字段说明：

- `targetCount`：默认生成几条提示词，前端限制为 1-20。
- `promptModel`：生成系列提示词使用的文本模型。
- `promptChannelId`：文本模型所属渠道。
- `promptInstruction`：系列拆分规则，例如封面、步骤图、总结图。
- `reviewRequired`：是否要求先审核提示词；关闭后会在草稿生成后自动触发生成。
- `concurrency`：批量生成时的并发上限，前端限制为 1-6。

### 提示词草稿

```ts
type SeriesPromptDraft = {
  id: string;
  title: string;
  prompt: string;
  status: "draft" | "running" | "success" | "failed";
  error?: string;
  resultIds?: string[];
};
```

草稿通过 `localforage` 缓存在浏览器本地，key 为 `infinite-canvas:series-drafts:{workflowId}`。运行中状态在重新加载时会回退为 `draft`，避免页面刷新后任务状态卡死。

## UI 设计

### 工作流列表

- 卡片一排最多展示 3 个。
- 卡片显示“单图/多图”模式标记。
- 顶部支持按分类筛选。
- 提供“AI 创建”“新建多图”“新建工作流”三个入口。

### 编辑弹窗

基础信息中可选择工作流模式：

- `单图生成`：沿用原有工作流。
- `多图生成`：显示“多图提示词规划”配置。

多图配置项：

- 提示词文本模型选择。
- 目标张数。
- 批量并发。
- 系列拆分说明。
- 是否先审核提示词。

图片生成模型仍使用“生成配置”里的图片模型。这样可以做到：文本模型负责规划提示词，图片模型负责最终出图。

### 运行弹窗

单图模式沿用原流程。

多图模式分为：

- 左侧：变量输入、参考图、生成提示词按钮。
- 右侧：基础提示词预览、生成参数摘要、多图提示词草稿列表。

草稿操作：

- 单条编辑标题和提示词。
- 单条复制、上移、下移、删除。
- 单条生成。
- 全部生成。
- 批量追加统一要求，例如“统一使用同一套品牌色和字体风格”。

## Agent 创建

工作流创建 Agent 的系统提示词已经要求识别多图场景。出现以下意图时优先创建 `multi_image_series`：

- 多张、系列、组图。
- 文章配图、步骤图、海报组。
- 写真组、方案集、装修方案。
- 为一篇帖子或文章生成多张配图。

Agent 输出 JSON 示例：

```json
{
  "mode": "multi_image_series",
  "seriesConfig": {
    "targetCount": "6",
    "promptInstruction": "按封面、痛点、功能、场景、对比、总结拆成 6 张图",
    "concurrency": "3",
    "reviewRequired": true
  }
}
```

后端会兼容 `variables` / `inputVariables`、`short_text` / `long_text` 等常见字段，降低 Agent 输出轻微偏差导致草稿失败的概率。

## 日志与排查

- 生成系列提示词使用文本 AI 调用日志。
- 图片生成使用图片 AI 调用日志。
- 管理后台 AI 调用日志拆分为“请求详情”和“返回详情”，可分别查看完整请求和完整返回。
- 日志文件仍按天写入本地 JSONL，不进入 SQLite，避免长期运行时数据库膨胀。

## 当前限制

- 草稿持久化仍是浏览器本地缓存，不随账号同步。
- “全部生成”是前端并发队列，刷新页面不会恢复未完成的网络请求。
- 参考图被多条提示词复用时仍遵循当前参考图生命周期策略：素材/生成结果不重复上传，本地临时图只在确认无用时物理删除。

## 后续方向

- 支持服务端任务队列，刷新页面后恢复批量生成任务。
- 支持草稿批量导入/导出。
- 支持按系列结果自动生成合集视图。
- 支持多图工作流运行统计和成本估算。
