# v0.3.0 审查与修复记录

审查日期：2026-05-31

## 审查范围

- 多图生成工作流新增功能。
- 工作流创建 Agent 对多图工作流的兼容。
- AI 调用日志前端页面与后端日志清洗逻辑。
- 生图参数中 Codex CLI 兼容模式的实际传参逻辑。
- React 重复 key、Ant Design 弃用告警、Go 启动 panic 和基础构建检查。

## 修复内容

### 多图生成工作流

- 新增 `multi_image_series` 工作流模式和 `seriesConfig` 配置。
- 增加“新建多图”入口和“小红书文章配图组”内置模板。
- 运行弹窗支持生成提示词草稿、逐条生成、全部生成、批量追加要求、上下移动、删除和失败重试。
- 草稿写入浏览器本地缓存，重新打开同一工作流时可恢复未完成草稿。
- 工作流结果继续写入生图历史，并按工作流名称自动归类。

### 工作流 Agent

- 后端标准化工作流草稿时兼容 `mode`、`seriesConfig` 和 `inputVariables`。
- 兼容 `short_text`、`long_text` 等变量类型别名，避免 Agent 输出轻微不一致导致保存失败。
- AI 创建工作流提示词已支持多图/系列/组图意图。

### AI 调用日志

- 修复 Go 正则中不合法 `{2048,}` 重复次数导致后端启动 panic 的问题。
- 日志清洗会提取 JSON、SSE、普通文本和 HTML 错误中的真实失败原因，降低 Base64、图片 URL 和代理噪声对排查的干扰。
- AI 调用日志页面把单个“详情”拆分为“请求详情”和“返回详情”，弹窗中完整显示内容，不再截断返回体。
- 本地直连日志上报开关采用乐观更新，保存失败时自动回滚。
- 优化后台 AI 调用日志顶部工具栏为单行布局。

### 生图参数与 UI 告警

- 修复 Responses 生图模式下 Codex CLI 兼容开关失效的问题：只有开启时才会追加兼容提示前缀。
- 修复多处标签列表使用 `tag` 作为 React key 的风险，避免重复标签触发重复 key 告警。
- 保持 `Drawer` 弃用属性清理结果，新增页面不再引入相关弃用用法。

- 生产构建需要在用户本机或服务器环境再次执行 `npm run build` 验证。
